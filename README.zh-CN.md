# GitSonar

[English](README.md) | [简体中文](README.zh-CN.md)

**这不是 Trending 查看工具，而是一个 GitHub 情报工作台。**

![GitSonar 主界面](assets/screenshots/trending.png)

GitSonar 是专为开发者、产品经理和研究人员打造的 Windows 桌面常驻应用，把“刷榜”变成一个可积累的**筛选 + 整理 + 跟踪 + 判断**的业务流。

## 三个典型场景

1. **去噪与信息沉淀**：把日常看到的仓库按 **关注、待看、已读、忽略** 分类，只看真正在乎的更新。
2. **跟进差异化更新**：不仅告诉你被关注仓库的 Star 涨了多少，更帮你追踪它们近期的 Release、Push 等硬核活跃变更。
3. **AI 加速辅助决策**：无需在网页间复制粘贴，在桌面一键将复杂仓库详情或多个竞品打包喂给 AI，直接获得结构化的商业/技术判断。

## 核心工作流

`发现 (Discover)` ➔ `整理 (Organise)` ➔ `跟踪 (Track)` ➔ `判断 (Judge)`

## 核心亮点

- **趋势发现**：今天 / 本周 / 本月热门仓库
- **状态管理**：关注、待看、已读、忽略
- **关注更新追踪**：Push、Star / Fork、Release 变化
- **仓库理解**：详情抽屉与 README 摘要
- **仓库对比**：双仓库并排比较
- **ChatGPT 提示词**：单仓库、批量、对比提示词
- **桌面常驻**：托盘、唤醒、关闭行为可配置、开机启动

## 它适合谁

- 想长期跟踪 GitHub 项目的开发者
- 做技术选型、竞品观察、产品研究的人
- 独立开发者和开源重度用户
- 想把“刷榜”变成“筛选 + 跟踪 + 判断”的人
- 更希望尽快形成判断，而不只是更快浏览的人

## 为什么它不是普通的 Trending 查看器

GitSonar 不只是帮你“看见项目”，而是帮你处理发现之后最耗时间的部分：

- 先发现，再判断
- 先标记，再处理
- 先关注，再跟踪
- 先对比，再理解
- 先分析，再决策

它更像一个长期运行的 GitHub 情报工作台，而不是一次性打开就结束的网页。

## 快速开始

### 下载与运行

**系统要求**：Windows 10 及以上系统。
**安装与更新**：目前暂无自动更新和代码签名，安装或运行时若遇到智能屏幕(SmartScreen)拦截，请选择“仍然运行”。

你可以在 [GitHub Releases](https://github.com/1wsslda/github-trend-radar/releases) 页面下载最新版本：

1. **安装版 (推荐)**：下载 `GitSonarSetup.exe` 进行安装。
2. **便携版 (便携)**：下载 `GitSonar.exe` 直接运行，即开即用。

第一次启动后，根据需要填写：
   - GitHub Token
   - 代理地址
   - 刷新间隔
   - 榜单数量
   - 关闭行为

### 推荐初始设置

- `GitHub Token`：有条件建议填写
- `代理地址`：网络不稳定时填写本地代理
- `刷新间隔`：建议 `1 小时`
- `榜单数量`：建议 `25`
- `关闭行为`：建议关闭主窗口时保留托盘运行

> **💡 什么是高阶研判中心？(下一步核心计划)**
> 这是把 GitSonar 推离普通排行榜的护城河。它将不只播报 “多了关注”，而是分析“最近 30 天最重要的变化是什么？和对比项目相比为何值得长线跟踪？”并提供结论。

## 后续优化方向 (Roadmap)

以下内容是后续演进方向，不代表当前已经实现。

### 第一优先：高阶更新研判中心 (深水区)

- **内嵌 AI 判断引擎**：支持 OpenAI、DeepSeek、Ollama 等协议，不只是“填装提示词”，而是独立输出结构化结论。
- **全要素变化感知**：按仓库聚合事件流、高活跃与大版本突出版次、自动划归更新优先级，及“自上次阅读以来的新东西”。

### 第二优先：建立专属长线情报池

- **私有流与过滤**：保存自定义的筛选器与视角，引入语言/类型过滤器。
- **账号打通**：支持从个人的 GitHub Account Stars 中首次自动导入关注库。
- **本地可靠性构建**：支持完善的网络联通性诊断及推荐值，提供安全的无损迁移与自动备份支持。

### 未来体验向

- 全局启动快捷键支持。
- 更完善的应用页面呈现与便携升级能力。

## 隐私与数据安全

对于一款涉及获取 GitHub 数据的桌面常驻工具，我们非常重视安全与信任边界：

- **高度本地化**：目前所有数据完全存储在一台机器的 `%LOCALAPPDATA%\GitSonar` 下。
- **Token 本地加密**：你自愿填写的 GitHub Token 只在使用 Windows 本机加密机制加密后存贴。
- **零外部收集**：除了与真实的 GitHub 接口、配置的 AI 客户端通讯以外，应用绝没有、也永远不包含采集你关注阅读习惯的其他远程嗅探行为。
