# GitHub Trend Radar

> 一个面向中文用户的 GitHub 趋势雷达桌面版。  
> 它不是“再做一个网页书签”，而是一套以本地服务为中枢、系统托盘为入口、桌面壳窗口为载体的轻量工作台。

## 项目简介

`GitHub Trend Radar` 用来解决一个很实际的问题：

很多人会看 GitHub Trending，但真正麻烦的从来不是“打开页面”，而是后面的工作流:

- 今天、本周、本月有哪些项目值得看
- 哪些仓库我要收藏、稍后再看、标记已读、直接忽略
- 收藏过的仓库最近有没有新版本、活跃度变化、重要更新
- 某个项目到底在做什么，值不值得继续跟
- 两个仓库看起来很像，到底谁更值得长期关注
- 我能不能一键把这些信息丢给 ChatGPT 做成中文分析

这个项目就是把上面这些动作，收进一个长期常驻、可关闭到托盘、对普通用户也友好的桌面应用里。

## 核心定位

从用户视角看，它是一个“GitHub 趋势 + 收藏追踪 + 中文分析 + 托盘常驻”的桌面工具。

从技术视角看，它不是传统 Electron 应用，而是：

- Python 负责抓取、缓存、状态管理、本地接口和托盘控制
- 本地 HTTP 服务负责把数据渲染成一套前端页面
- Chrome / Edge 的应用模式窗口负责显示主界面
- 托盘图标负责常驻、唤醒、刷新和退出

这种方案的好处是：

- 界面灵活，更新快
- 包体和运行形态更轻
- 不需要完整维护一套 Electron 工程
- 对“长期开着、偶尔点开看看”的工具非常合适

## 核心功能

- `趋势看板`：同时展示 `今天 / 本周 / 本月` 的 GitHub 热门仓库
- `双源聚合`：综合 GitHub Trending 页面和 GitHub API 的数据
- `中文信息增强`：仓库简介、README 摘要、详情信息更适合中文阅读
- `仓库详情`：查看 Stars、Forks、License、最近更新时间、README 摘要等
- `仓库对比`：选中两个仓库后直接做并列对比
- `ChatGPT 分析`：支持单仓库、批量仓库、仓库对比的提示词生成与打开
- `状态管理`：支持 `收藏 / 稍后看 / 已读 / 忽略`
- `收藏更新追踪`：持续检测收藏仓库的新变化，集中显示在“收藏更新”面板
- `导出数据`：导出用户状态，便于备份和迁移
- `系统托盘常驻`：支持隐藏到托盘、托盘唤醒、托盘刷新、托盘退出
- `关闭行为可配置`：关闭主窗口时可选择“保留托盘运行”或“直接退出程序”
- `开机启动`：支持 Windows 开机自启，并避免额外控制台窗口干扰
- `代理支持`：支持手动填写代理地址，也会尝试识别常见本地代理端口

## 适合谁

- 想长期跟踪 GitHub 热门项目的人
- 想把“发现项目”变成“筛选项目 + 跟踪项目”的人
- 做产品、独立开发、技术选型、竞品观察的人
- 喜欢把仓库先分类，再集中处理的人
- 想把 GitHub 内容快速转成中文分析结论的人

如果你只是偶尔打开一下 Trending 页面，这个工具可能有点重。  
如果你每天都要反复看 GitHub、收藏项目、做判断，它会非常顺手。

## 运行效果

应用主窗口提供一套桌面化工作台：

- 顶部是全局操作区：排序、分析、刷新、隐藏到托盘、设置、导出
- 中间是多面板内容区：今天 / 本周 / 本月 / 收藏 / 稍后看 / 已读 / 忽略 / 收藏更新
- 每张卡片都可以直接做状态切换、打开详情或发给 ChatGPT 分析
- 关闭主窗口后，程序可根据设置继续在托盘运行

这意味着它更像一个“GitHub 情报台”，而不是一个单次打开就结束的网页。

## 快速开始

### 给普通用户

如果你只是想直接使用，不关心源码：

1. 安装 `dist/installer/GitHubTrendRadarSetup.exe`
2. 或者直接运行 `dist/GitHubTrendRadar.exe`
3. 第一次启动后，按需要在“设置”里填写：
   - GitHub Token
   - 代理地址
   - 刷新间隔
   - 榜单条数
   - 关闭行为

### 给第一次使用的人一个建议

建议先这样配置：

- `GitHub Token`：有就填，没有也能用
- `代理地址`：如果你的网络访问 GitHub 不稳定，填本地代理地址
- `刷新间隔`：建议 `1 小时`
- `榜单条数`：建议 `25`
- `关闭行为`：建议先选 `关闭主窗口时保留托盘运行`

这样体验最稳定，也更符合它“常驻雷达”的定位。

## 网络与 VPN 说明

这个项目 `不内置` VPN、翻墙或网络加速能力。

要点很简单：

- 如果你当前网络本来就能正常访问 GitHub，那么不需要 VPN
- 如果你当前网络打不开 GitHub、API 超时、详情加载失败，那么你需要先准备可用网络环境
- 程序支持填写代理地址，但它不会替你创建代理

常见场景：

- Clash / Mihomo 用户：通常填写 `http://127.0.0.1:7890`
- 其他本地代理：填写你自己的本地端口
- 不确定时：先确认浏览器里能正常打开 GitHub，再回来使用这个程序

## GitHub Token 有什么用

不填 Token 也可以用，但会有两个现实差异：

- 请求频率更容易受 GitHub 限流影响
- 收藏更新追踪会更保守，刷新频率更低

填入 Token 后，程序在以下方面通常会更稳定：

- 仓库详情获取
- 收藏仓库更新追踪
- 持续使用时的 API 成功率

在 Windows 上，Token 会通过 `DPAPI` 做本地加密保存，仅当前用户环境可解密；它不是上传到外部服务器的云配置。

## 使用说明

### 1. 趋势面板

默认提供三个时间范围：

- `今天`
- `本周`
- `本月`

每个面板里的仓库可以按不同方式排序：

- 总星标
- Trending 趋势
- 增长量
- Fork 数
- 仓库名
- 语言

### 2. 状态管理

每个仓库都可以被标记为：

- `收藏`
- `稍后看`
- `已读`
- `忽略`

这些状态会长期保存在本地，方便你把“看见一个仓库”变成“处理这个仓库”。

### 3. 收藏更新

这是整个项目里非常实用的一块。

当你把仓库加入收藏后，程序会按策略定期检查这些仓库的变化，例如：

- 最近推送时间变化
- Star / Fork 变化
- Release 标签变化

有新变化时，它会出现在 `收藏更新` 面板里，避免你手动一个个翻。

### 4. ChatGPT 分析

程序支持三类分析入口：

- `单仓库分析`
- `当前列表批量分析`
- `两个仓库对比分析`

并提供三种输出方式：

- 打开 `ChatGPT 网页版`
- 打开 `ChatGPT 桌面版`
- 仅复制提示词

这部分的设计不是“把 AI 硬塞进去”，而是把原本你自己会做的提示词整理工作自动化。

### 5. 关闭主窗口与托盘

现在支持两种关闭行为：

- `关闭主窗口时保留托盘运行`
- `关闭主窗口时直接退出程序`

如果选择保留托盘：

- 关掉主窗口后，程序不会退出
- 你可以从系统托盘重新打开
- 托盘图标可能收在任务栏右下角的隐藏图标里

如果你看到“程序已经在运行”，通常就是它还在托盘里。

## 设置项说明

| 设置项 | 作用 | 建议 |
| --- | --- | --- |
| `GitHub Token` | 提高 API 稳定性，减少限流影响 | 有条件建议填写 |
| `代理地址` | 用于访问 GitHub 或 API | 网络不稳定时填写 |
| `刷新间隔（小时）` | 控制后台刷新频率 | `1` 或 `2` 小时 |
| `榜单条数` | 每个面板显示的仓库数量 | `25` 比较均衡 |
| `端口` | 本地服务监听端口 | 一般保持默认 |
| `关闭行为` | 关闭主窗口时保留托盘或直接退出 | 推荐先选 `tray` |
| `开机启动` | 开机后自动启动程序 | 想长期常驻时开启 |

## 技术架构

可以把它理解成下面这条链路：

```text
GitHub Trending 页面 + GitHub API
                ↓
       Python Runtime（抓取 / 翻译 / 缓存 / 状态）
                ↓
        Local HTTP Server（127.0.0.1）
                ↓
     桌面壳窗口（Chrome / Edge App Mode）
                ↓
         System Tray（托盘常驻 / 唤醒 / 退出）
```

这套结构的关键特点：

- `单实例运行`：重复打开时会尽量唤醒已有实例，而不是堆出多个窗口
- `本地缓存回退`：网络波动时尽量保留上次数据，不直接空白
- `托盘优先`：更适合“长期挂着”的工具型应用
- `GUI 子系统打包`：避免正常运行时出现多余控制台窗口
- `本地数据持久化`：用户状态、设置、缓存都保存在本机

## 本地数据与存储位置

打包后的程序会把运行数据放在：

```text
%LOCALAPPDATA%\GitHubTrendRadar
```

常见文件包括：

- `settings.json`：设置项
- `user_state.json`：收藏、稍后看、已读、忽略等用户状态
- `data/latest.json`：最近一次抓取的趋势快照
- `repo_details_cache.json`：仓库详情缓存
- `.translation_cache.json`：翻译缓存
- `runtime_state.json`：当前实例运行状态

源码方式运行时，这些文件默认会落在项目目录本身。

## 常见问题

### 1. 为什么点了关闭，程序还说“已经在运行”？

因为你当前设置的可能是“关闭主窗口时保留托盘运行”。  
这不是 bug，而是它的常驻工作方式。

处理方式：

- 到任务栏右下角找托盘图标
- 或在设置里把关闭行为改成“直接退出程序”

### 2. 为什么我没看到托盘图标？

Windows 经常会把托盘图标折叠到隐藏区域。  
请点任务栏右下角的小箭头展开隐藏图标。

### 3. 为什么趋势列表刷不出来？

通常不是程序坏了，而是网络访问 GitHub 不通。

优先检查：

- 浏览器能不能打开 `github.com`
- 代理是否已启动
- 程序里的代理地址是否填写正确
- GitHub API 是否被限流

### 4. 不填 Token 能用吗？

能用，但更容易受到限流影响。  
如果你会长期使用，建议配置自己的 GitHub Token。

### 5. 为什么它像桌面应用，但又有网页界面？

因为它本质上是“本地服务 + 桌面壳”的结构。  
这样做的目的不是炫技，而是为了兼顾：

- 开发效率
- UI 灵活度
- 托盘常驻能力
- 打包部署便利性

### 6. 双击 `.ps1` 一闪而过怎么办？

直接用：

```cmd
build_all_click.cmd
```

这个脚本会停留在窗口里，不会一闪而过。
