## 标题

`代码签名`

## 摘要

- 这个任务目标是为 Windows 发布产物建立代码签名信任路径。
- 现在不能直接实现，因为真实签名需要证书、私钥、时间戳服务和发布账号决策。
- 本轮安全处理方式是记录阻塞状态和未来恢复任务所需的外部依赖，不触碰任何密钥或证书。

## 目标

- 主要目标：记录代码签名当前阻塞原因。
- 次要目标：明确未来恢复此任务前必须由用户或发布者提供的安全材料和决策。
- 成功标准：
  - 任务状态标记为 `[!]`。
  - 阻塞原因写入计划、任务表、Sprint 队列和进度日志。
  - 不新增签名脚本、不读取证书、不使用时间戳服务。

## 非目标

- 不生成自签名证书。
- 不保存 PFX、私钥或密码。
- 不调用 `signtool.exe`。
- 不修改打包脚本。
- 不改变发布流程。

## 用户影响

- 用户可见功能无变化。
- 发布者仍可使用现有 SHA256 manifest 辅助校验。

- 阻塞原因：当前没有可用证书、私钥保管方案、密码注入方式、时间戳服务和 CI/本地签名边界。
- 未来实现必须确保签名凭据不写入仓库、不写入日志、不通过普通设置保存。
