# FAQ

## 为什么我关掉窗口后，程序还提示“已经在运行”？

通常是因为当前关闭行为设置成了“关闭主窗口时保留托盘运行”。

这种情况下：

- 主窗口关闭后程序不会退出
- 你可以从系统托盘重新唤醒它
- 再次双击程序时，会优先尝试唤醒已有实例

如果你希望点关闭就彻底退出，请到“设置”里把关闭行为改成“关闭主窗口时直接退出程序”。

## 为什么我没看到托盘图标？

Windows 经常会把托盘图标收进任务栏右下角的隐藏图标区域。  
如果主窗口已经关掉，但程序仍在运行，请先检查隐藏图标区。

## 使用这个程序必须开 VPN 吗？

不一定。

- 如果你的网络本来就能正常访问 GitHub，通常不需要 VPN
- 如果当前网络无法访问 GitHub、API 超时、详情加载失败，就需要先准备好可用网络环境

GitSonar 支持填写代理地址，但**不提供 VPN / 翻墙 / 加速能力本身**。

## 为什么有些仓库的增长不是实时的？

趋势来源并不完全一致。

- 来自 GitHub Trending 的条目，通常能拿到更直接的趋势增长信息
- 仅来自 GitHub API 的条目，原始接口不会直接给出“今天增长多少”

当前产品会尽量结合可用来源和本地历史快照做展示，但并不是每个仓库都能在第一次出现时立刻得到完整增长基线。

## 不填 GitHub Token 能用吗？

可以用，但会有差别：

- API 更容易遇到限流
- 收藏更新追踪会更保守
- 仓库详情获取的稳定性会差一些

如果你会长期使用，建议填写 Token。

## Token 会不会上传到外部服务器？

不会。

Token 只会用于访问 GitHub API，并在 Windows 本地使用 DPAPI 加密保存。  
它不是云配置，也不会被上传到第三方服务器。

更多说明见 [SECURITY.md](SECURITY.md)。

## 打包时为什么会出现命令窗口？

打包脚本本身就是通过 PowerShell 运行的，所以会打开命令窗口显示日志。  
如果你直接用 `scripts/build_all_click.cmd`，窗口会在执行结束后停住，方便查看成功或失败原因。

## 为什么我电脑里还看得到 `GitHubTrendRadar` 旧目录？

这是品牌迁移后的兼容痕迹。

现在新的默认运行目录已经是：

- `%LOCALAPPDATA%\GitSonar`

但如果你之前用过旧版本，电脑里可能还会保留：

- `%LOCALAPPDATA%\GitHubTrendRadar`

新版本启动时会自动尝试把旧目录中的配置、状态和缓存合并到新目录。旧目录是否继续保留，不会影响新版本正常使用；它更像一个迁移来源和历史备份。
